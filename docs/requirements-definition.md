# **Shukan (就活管理サービス) 要件定義書（第2.0版）**

## **1. はじめに**

### **1.1. 文書の目的**

本書は、新規開発する就活管理サービス「Shukan」のシステム化要件を定義し、開発チームおよび関係者間での仕様認識を統一することを目的とする。

### **1.2. システムの目的と背景**

現在の日本の就職活動において、ユーザーは複数の就活サービスを併用することが一般的となっており、情報の分散や大量のメールによる「管理作業の負担」が課題となっている。

「Shukan」は、ユーザーのメールボックス（Gmail）と連携し、AIを活用してメール内容から就活情報を自動抽出・一元管理することで、ユーザーが「就活のスケジュールや状況管理」という本質的ではない作業から解放され、就職活動そのものに集中できる環境を提供することを目的とする。

### **1.3. 適用範囲（スコープ）とフェーズ**

- **対象ユーザー:** 日本で新卒就活を行うユーザー
- **対象メールプロバイダ:** Gmailのみ（他プロバイダは非対応とする）
- **対象プラットフォーム:** Webアプリケーション（スマホ表示に最適化されたレスポンシブWebアプリ）

## **2. システムの全体像・業務要件**

### **2.1. 業務フローの変化**

- **As-Is（現状）:** ユーザーは複数のサイトへ個別にログインし、大量の受信メールを自力で仕分け、手動でスケジュール、選考状況、タスクを管理・把握している。
- **To-Be（導入後）:**
  1. ユーザーはFirebase Authenticationでログインし、Gmailの読み取りを許可する。
  2. システムが過去のメールをスキャンし、進行中の就活情報を自動で取り込む。
  3. その後、バックエンドシステムが就活関連のメールをリアルタイムに検知し、Vertex AI（Gemini）で解析して企業名・イベント日程・タスク等を抽出する。
  4. 解析結果やエラーはFCM（Web Push）経由でリアルタイムにWebアプリに同期・通知される。
  5. ユーザーはダッシュボード上で「今やるべきこと」や「直近の予定」を即座に把握し、アクションに集中できる。

## **3. 機能要件**

### **3.1. ユーザー認証・セキュリティ機能**

- **認証基盤:** Firebase Authentication を利用したログイン機能。OAuth 2.0を利用し、Gmailへの安全なアクセス権限（Restricted Scope）を取得する。
- **トークン管理:** 取得したOAuthトークンは、Cloud Key Management Service (KMS) を用いて暗号化され、データベース（Neon）に安全に保存される。
- **アクセス制御:** Cloudflare CDNおよびFirebase App Checkを利用し、不正なアクセスをブロックする。

### **3.2. AI活用型メール連携・自動解析機能（システムのコア）**

- **リアルタイム検知:** Gmail APIからの更新をトリガーに、バックエンドが新着メールを取得する。
- **AI情報抽出（パース）:** 取得したメール内容を Vertex AI (Gemini 2.5 Flash) に入力し、JSON形式の構造化データ（企業名、選考状況、タスク、イベント日時など）として抽出する。

### **3.3. 共通UI・ナビゲーション（Webアプリ）**

- **レイアウト:** 左上にロゴ・サービス名、左側にサイドバーナビゲーション、中央上部に検索バーを配置する。
- **右上のメニュー:** アカウントロゴ・メニューと「通知アイコン」を配置する。
- **通知機能:** 運営からのお知らせや、バックエンドでのメール解析・処理失敗時のエラー通知を通知アイコンにて受け取る。

### **3.4. ダッシュボード機能（Webアプリ）**

「今日何をすべきか」をひと目で把握できるサマリー画面。

- **各種サマリー:**
  - 進行中の企業数（企業一覧へ遷移可能）
  - 未完了のタスク数（1週間以内、タスク一覧へ遷移可能）
  - イベント数（1週間以内、イベント一覧へ遷移可能）
  - 未読スカウト数（スカウト一覧へ遷移可能）
  - タスク完了率
  - 最終更新日時
- **直近の詳細表示:** 直近のタスク詳細、イベント詳細、スカウト詳細を表示する。
- **要対応アラート:** 以下の条件に合致する場合、優先的にアラートを表示する。
  - 24時間以内に対応が必要なタスク
  - 2日以上返信していないメール（※土日祝日はカウントから除く）
  - バックエンド処理の失敗通知

### **3.5. 企業管理機能（Webアプリ）**

- **リスト表示:** 企業名、企業カラー、応募職種、現在のステータス（選考前、選考中、内定済、終了済など）をリスト形式で表示する。
- **企業詳細（ダイアログ）:** リストから項目を選択すると、以下の詳細情報をダイアログで表示する。
  - 基本情報（企業カラー、企業名、応募職種、ステータス）
  - 直近のアクション（直近のタスク、直近のイベント）
  - 詳細情報（対応就活サービス名、連絡先メールアドレス、マイページURLなど）
- **操作機能:** 企業名での検索、タスク・イベントの日程順ソート、ステータスでのフィルタリング。
- **手動追加・修正:** 企業情報の追加および修正が可能。入力項目は「企業名」「応募職種（例：フロントエンドエンジニア、SRE等）」「連絡先」「対応サービス」など。

### **3.6. タスク管理機能（Webアプリ）**

- **タスク一覧表示:** 全企業にまたがるタスクを一覧表示する。各タスクには「タスク種別（メール返信、書類送信等）」「紐づく企業名」「期限」が表示される。
- **状態管理:** 未完了 / 完了済みのトグル切り替えによる表示の分離。
- **操作機能:** タスク種別でのフィルタリング、手動でのタスク追加・編集機能。

### **3.7. イベント管理機能（Webアプリ）**

- **イベント一覧表示:** 説明会、人事面談、ハッカソン、面接などの各種イベントを一覧表示する。
- **表示内容:** イベント種別、紐づいている企業名（存在する場合）、イベント日時。
- **状態管理:** イベント日時に基づき、「開催前」「開催中」「開催終了」で表示を分割する。ユーザーはイベントの「日時」や「タイトル」等の詳細を編集できるが、「開催ステータス」自体はシステムが日時に基づき自動判定するため、手動での変更は不可とする。
- **操作機能:** イベント種別でのフィルタリング、手動でのイベント追加・編集機能。

### **3.8. スカウト一覧管理機能（Webアプリ）**

- **スカウト一覧表示:** スカウト種別（説明会、インターン等）、スカウト元の企業名、スカウト場所のサービス（サポーターズ等）を一覧表示する。
- **状態管理:** 「対応済み（承諾・辞退など）」と「未対応」で表示を分割する。承諾・辞退・未対応のトグルで切り替え可能とし、この操作をもって不要なスカウトの整理（アーカイブの代替）とする。

### **3.9. リアルタイム同期機能（FCM）**

- **Web Push連携:** OSレベルのプッシュ通知はMVPスコープ外とするが、Webアプリを開いている間の「UIのリアルタイム更新」および「未読バッジの更新」を実現するため、Firebase Cloud Messaging (FCM) と Pub/Sub を活用する。これにより、バックエンドでのAI解析完了やエラー発生を即座に画面へ反映させる。

## **4. 非機能要件**

- **可用性・性能:** Serverlessアーキテクチャ（Cloud Run, Neon）によるトラフィック追従。Redis (Upstash) を活用したキャッシュ・セッション管理による高速化。
- **非同期フィードバック:** バックエンドでのAI解析処理の完了やエラーは、Firebase Cloud Messaging (FCM) と Pub/Sub を連携させ、フロントエンドへリアルタイムに通知する。
- **セキュリティ:** Firebase App Check によるリクエスト元の検証、Secret Manager を用いたカスタム認証キーの管理により、APIサーバー（Cloud Run）への直接アクセスを防ぐ。
- **IP制限:** Cloudflareからのリクエスト（CF-Connecting-IP）を検証し、許可されたIP以外からのアクセスを遮断する。

## **5. システムアーキテクチャ構成**

- **フロントエンド:** Web Application (React)
- **CDN / セキュリティ:** Cloudflare CDN
- **認証・セッション:** Firebase Authentication
- **バックエンドAPI:** Cloud Run (Go)
- **データベース (RDB):** Neon (PostgreSQL)
- **データベース (KVS):** Upstash (Redis)
- **AI・LLM:** Vertex AI (Gemini 2.5 Flash)
- **メール連携:** Gmail API (テスト環境) / OAuth Authorization Server
- **暗号化・シークレット管理:** Google Cloud Key Management Service (KMS), Secret Manager
- **リアルタイム通知:** Firebase Cloud Messaging (FCM), Google Cloud Pub/Sub
- **監視・ロギング:** Cloud Logging, Cloud Monitoring
