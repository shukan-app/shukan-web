# **Shukan (就活管理サービス) 要件定義書（第1.6版）**

## **1\. はじめに**

### **1.1. 文書の目的**

本書は、新規開発する就活管理サービス「Shukan」のシステム化要件を定義し、開発チームおよび関係者間での仕様認識を統一することを目的とする。

### **1.2. システムの目的と背景**

現在の日本の就職活動において、ユーザーは複数の就活サービスを併用することが一般的となっており、情報の分散や大量のメールによる「管理作業の負担」が課題となっている。

「Shukan」は、ユーザーのメールボックス（Gmail）と連携し、AIを活用してメール内容から就活情報を自動抽出・一元管理することで、ユーザーが「就活のスケジュールや状況管理」という本質的ではない作業から解放され、就職活動そのものに集中できる環境を提供することを目的とする。

### **1.3. 適用範囲（スコープ）とフェーズ**

- **対象ユーザー:** 日本で新卒就活を行うユーザー
- **対象メールプロバイダ:** Gmailのみ（他プロバイダは非対応とする）
- **フェーズ1（MVP対象）:** Webアプリケーション（スマホ表示に最適化されたレスポンシブWebアプリ）
- **フェーズ2以降:** Android/iOSネイティブアプリケーションの展開、プッシュ通知の導入

## **2\. システムの全体像・業務要件**

### **2.1. 業務フローの変化**

- **As-Is（現状）:** ユーザーは複数のサイトへ個別にログインし、大量の受信メールを自力で仕分け、手動でスケジュール、選考状況、タスクを管理・把握している。
- **To-Be（導入後）:**
  1. ユーザーはGoogleアカウントでWebアプリにログインし、Gmailの読み取りを許可する。
  2. 登録直後、システムが過去のメールをスキャンし、すでに進行中の就活情報を自動で取り込む。
  3. その後、バックエンドシステムが就活関連のメールをリアルタイムに検知し、AIで解析して企業名・選考状況・**必要なタスクとその期限**を抽出する。
  4. ユーザーはアプリを開くだけで、整理された状況と「今やるべきタスク」を把握できる。完了したタスクはチェックをつけて管理する。

## **3\. 機能要件**

### **3.1. ユーザー認証機能**

- **シングルサインオン（SSO）:** Google Cloud Identity Platform を利用した会員登録・ログイン機能。ログイン時にOAuth 2.0を利用し、Gmailへの安全なアクセス権限（Restricted Scope）を取得する。

### **3.2. AI活用型メール連携・自動解析機能（システムのコア）**

- **初期自動同期（初回スキャン）:** ユーザーの新規登録時、Gmailの受信履歴をバッチ処理でスキャンし、就活に関連する過去のメールを抽出・自動解析する。
- **リアルタイムメール検知:** Gmail APIのPush通知機能（Google Cloud Pub/Sub）を利用し、新着メールの受信をリアルタイムに検知・取得する。
- **事前フィルタリング（コスト対策）:** 送信元ドメインや件名のキーワードを用いて、プログラム側で「明らかに就活関連のメール」のみを一次選別する。
- **AI情報抽出（パース）:** フィルタリングを通過したメールをクラウドLLM（Gemini 3 Flash）に入力し、「企業名」「メールの性質」「選考状況」、および「ユーザーが実行すべきタスクとその期限（RFC 3339形式）」を構造化データとして抽出・データベースへ登録する。

### **3.3. ダッシュボード・企業管理機能（Webアプリ）**

現在就活を進めている企業の状況を一元管理する。

- **企業情報一覧表示:** AIが抽出した情報を一覧表示。表示形式は「カード形式」と「リスト形式」を切り替え可能。
  - **期限の自動算出:** 企業ごとのアクション期限は、その企業に紐づく「未完了タスク」の中で最も直近の期限を自動算出して表示する。
- **検索・ソート・フィルタリング機能:** フリーワード検索（企業名・タイトル）、期限ソート、ステータス絞り込み。
- **手動追加・編集機能:** ユーザー自身で企業情報（企業名・ステータスのみのシンプル入力）を追加・修正できる機能。タスクは別途追加する。

### **3.4. タスク管理機能（Webアプリ）**

企業ごとの選考に必要なタスクを管理する。各タスクは固有の「期限」を持つ。

- **全タスク一覧表示:** アプリ下部のタブから、全企業にまたがるタスクを期限順で一覧表示する画面へアクセス可能。
- **ステータス（完了/未完了）管理:** タスクのチェックボックスをタップすることで、「完了済み」状態へ移行できる。
  - 完了済みタスクは専用のセクションに表示され、チェックを外すことで「未完了」へ差し戻すことができる。
- **AI自動生成タスクの視覚的区別:** AIが自動抽出したタスクには専用バッジ（「✨AI」など）を表示し、手動追加タスクと区別する。
- **手動タスク管理:** ユーザー自身が「対象企業」「タスク名」「期限」を指定して任意のタスクを追加・編集・削除できる。

### **3.5. スカウト一覧・整理機能（Webアプリ）**

- **未読・既読管理:** 新着スカウトは未読状態として扱い、アプリのナビゲーションバー等に未読件数バッジを表示する。
- **一覧・タブ切り替え機能:** 「受信トレイ（有効なスカウト）」と「アーカイブ（整理済みスカウト）」をタブで切り替えて表示する。
- **アーカイブ・復元機能:** 不要なスカウトを「アーカイブ」へ移動し、受信トレイを整理する機能。誤ってアーカイブしたものを元に戻す（復元する）ことも可能。
- **詳細遷移機能:** スカウトメール本文に記載されているマイページやイベント登録用URLへ、別タブ（ブラウザ）で遷移する機能。

### **3.6. 外部通知（プッシュ通知）機能**

- **フェーズ1（MVP）では実装しない:** Webブラウザを開いた際のアプリアイコンバッジや、アプリ内通知でのみ情報を伝達する。
- ※フェーズ2以降のAndroid/iOSアプリ展開時に、FCMを用いたプッシュ通知（リマインド等）を実装する。

### **3.7. システム内通知機能（Webアプリ）**

- **エラー通知（抽出失敗時のリカバリー）:** 抽出失敗時、「手動での対応をお願いいたします。」等の文言と識別情報を表示。「未対応（警告色）」と「対応済み（非活性化色）」のステータス管理を行い、未対応エラーからは企業情報の手動追加画面へ直接遷移できる。
- **運営からのお知らせ:** メンテナンス情報等をリスト形式で表示。

## **4\. 非機能要件**

- **可用性・性能:** サーバーレス構成によるトラフィック追従。
- **セキュリティ・プライバシー:** Gmail読み取り（Restricted Scope）に関する厳格なセキュリティ実装、プライバシーポリシー策定。Identity Platformによるセキュアな認証。
- **インフラ・運用コスト:** 初回スキャン時は対象メールが大量になるため、バックグラウンドのバッチ処理（Cloud Tasks等のキューイング）により負荷を分散し、APIのレートリミット超過とコストスパイクを防ぐ。

## **5\. 制約事項・スコープ外（作らないもの）**

- **対象外のメールプロバイダ:** Gmail以外のメール連携は実装しない。
- **外部カレンダー連携（Googleカレンダー等）:** 本サービス単体で管理を完結させるため実装しない。
- **AIチャット機能:** 実装しない。

## **6\. システムアーキテクチャ構成**

- **フロントエンド:** React, Tailwind CSS
- **ホスティング:** Firebase Hosting または Vercel
- **認証基盤:** Google Cloud Identity Platform (OAuth 2.0)
- **バックエンドAPI:** Cloud Run functions (TypeScript, Node.js)
- **データベース:** Neon (Serverless PostgreSQL)
- **LLMモデル:** Gemini 3 Flash
- **時刻フォーマット:** RFC 3339形式
