# Workflow

## 基本原則

- 機能追加・修正は 計画 → 実装 → リンター実行 → テスト の4段階で進めること。

## 計画

- 曖昧な点は有耶無耶にせず、必ず質問すること。

## 実装

- 実装は ドメイン層 → アプリケーション層 → インフラ層 → プレゼンテーション層 の順番で進めること。
- 詳細は `./docs/architecture.rules.md` を参照すること。

## リンター実行

- 定義されているスキル `lint` を必ず実行すること

## テスト

- 単体テストは `Vitest` を用いて実施すること。
- 詳細は `./docs/tests.rules.md` を参照すること。

## Communication

- 会話は日本語で回答すること。
